# Phase 1 课程学习设计：基础操作与生存

> 基于 2026-07-01 讨论定稿

## 一、三阶段学习策略总览

| 阶段 | 名称 | 目标 |
|------|------|------|
| **Stage 1** | 基础操作与生存（本文档） | 学会走中线、推进、放炸弹炸砖、基本 Buff 拾取 |
| **Stage 2** | 生成对抗式轨迹模仿（GAIL） | 人类 vs Stage 1 AI 对战采集轨迹 → BC 预热 → GAIL 精调 |
| **Stage 3** | 自主双智能体对战（稀疏奖励） | 自博弈，+1/-1/0 稀疏奖励最终优化 |

### 设计原则

- **每阶段的奖励函数独立**，仅用于学习该阶段目标
- **子阶段过渡**：前一阶段奖励项权重减半保留，新奖励项以显式权重加入
- **对手**：每轮随机初始化位置，每轮内固定不动

- F: 地板（奇-奇坐标）
- S: 石墙（偶-偶坐标，不可破坏）
- ?: 砖块（概率 0.7）或地板（奇-偶/偶-奇坐标）
- 尺寸：19×11（默认），CELL_SIZE=40px，碰撞盒=32px（0.8×），每侧 4px 容差

### 中线偏离计算

玩家当前所在格子 (gx, gy)：
- 该格子的中心像素坐标 (cx, cy) 由 `grid_center` 得到
- 走廊方向判断：
  - 若 gy 为奇数（水平走廊），中线为 y = cy，偏离距离 = |pos_y - cy|
  - 若 gx 为奇数（垂直走廊），中线为 x = cx，偏离距离 = |pos_x - cx|
  - 若 gx、gy 均为奇数（交叉口），偏离距离 = min(|pos_x - cx|, |pos_y - cy|)
  - gx、gy 均为偶数的格子（石墙）不会出现
- 归一化尺度：half_cell = CELL_SIZE / 2 = 20px

## 三、Phase 1 子阶段设计

### Phase 1.1：纯走廊生存（~10K 步）

**地图配置**：`BRICK_GEN_PROB = 0`，全场无砖块，仅奇-奇走廊+石柱

**目标**：学会沿走廊中线行进，持续朝向对手位置推进

| 类 | 奖励项 | 值 | 说明 |
|----|--------|----|------|
| 🟢 | 曼哈顿距缩小（窗口平均） | +0.1 / 格 | 每 10 帧计算窗口平均位置，奖励该平均值相对对手的缩小 |
| 🔴 | 曼哈顿距增大 | -0.02 / 格 | 远离对手则轻量惩罚 |
| 🔴 | 中线偏离惩罚 | -0.04 × (dist/20)² | 偏离走廊中心线，二次型加重偏大值 |
| 🔴 | 连续 30 帧（≈1.25s）无净推进 | -0.02/帧，叠至 -0.5 上限 | 防止原地徘徊或来回走；推进判定为曼哈顿距缩小 >0 |
| 🔴 | 撞墙 | -0.03/轴 | 有方向输入但坐标未变 |
| 🔴 | 撞对手 | -0.02 | 被对方阻挡 |
| 🟢 | 存活 | +0.001/帧 | 每帧基本分 |
| 🔴 | 非法操作-炸弹上限 | -0.1 | action=1 但已达 bomb_max |
| 🔴 | 非法操作-脚下不可放弹 | -0.05 | action=1 但脚下非 floor |
| 🔴 | 非法操作-无效 ignite | -0.05 | ignite=1 但无遥控能力或队列空 |
| 🔴 | 非法操作-超 2 方向 | -0.05 | count>2 |
| 🔴 | 被自己炸弹炸死 | -1.0 | 区分于对手 |
| 🔴 | 被对手炸弹炸死 | -0.5 |  |

### Phase 1.2：砖块爆破与开路（~50K 步）

**地图配置**：恢复 `BRICK_GEN_PROB = 0.7`

**新增奖励**（1.1 所有项权重减半保留）：

| 类 | 奖励项 | 值 | 说明 |
|----|--------|----|------|
| 🟢 | 成功放置炸弹 | +0.1 | action=1 且合法放置 |
| 🟢 | 自己炸弹炸毁**向对手路上的砖** | +0.5/块 | 非一次性检测（见下方方向判定） |
| 🟢 | **杀死对手** | **+4.0** | 对手在本帧死亡且 Agent 存活 |
| 🔴 | 炸弹爆炸但 **0 砖块被毁** | -0.2 | 鼓励精准放置，减少浪费 |

**"向对手路上"方向判定**：

```python
# 炸弹爆炸时，对于每块被摧毁的砖块 (bx, by)
bomb_gx, bomb_gy = bomb的网格位置
opp_gx, opp_gy = opponent当前的网格位置

# 从炸弹指向对手的方向向量
dir_to_opp = (opp_gx - bomb_gx, opp_gy - bomb_gy)
# 从炸弹指向砖块的方向向量
dir_to_brick = (bx - bomb_gx, by - bomb_gy)

# 点积 > 0 → 砖块在对手方向侧 → 全奖励
# 点积 ≤ 0 → 砖块在反方向或侧向 → 仅 0.1（副产物鼓励）
dot = dir_to_opp[0] * dir_to_brick[0] + dir_to_opp[1] * dir_to_brick[1]
reward = 0.5 if dot > 0 else 0.1
```

砖块总量约 70 块，一次性消耗。全部炸完后 Agent 不再获得砖块奖励，自然转向对手推进。**炸弹空爆惩罚**防止 Agent 在已清空区域无脑放弹。

### Phase 1.3：Buff 引入（~30K 步）

**新增奖励**（1.1+1.2 所有项权重减半保留）：

| 类 | 奖励项 | 值 | 说明 |
|----|--------|----|------|
| 🟢 | 拾取任意 Buff | +0.2 | 所有类型 |
| 🟢 | 拾取未知道具 | +0.3 | 区别于永久增强 |

### Phase 1.4：融合评估（~10K 步）

不做训练，仅评估。在最终环境配置下评测：

- 平均生存帧数
- 场均炸弹数 & 场均有效炸砖数
- 胜率 vs 随机对手（100 局）
- 场均非法操作数
- 中线偏离均值

## 四、奖励函数架构

### 接口定义

使用现有的 `rewards/` 架构，`RewardFunction` 子类接收：

```python
class Phase1Reward(RewardFunction):
    def __call__(self, engine, prev_snap, snap, action, agent_id):
        ...
        return reward
```

### 非法操作检测

`engine.step()` 中已经包含了动作的静默忽略逻辑。Reward 函数通过对比 `prev_snap` 和 `snap` 中的玩家状态来推算操作是否有效：

| 非法操作 | 检测方式 |
|----------|----------|
| 已达上限放炸弹 | action["action"]=1 且 bomb_placed_count 未增加 |
| 脚下非法放炸弹 | action["action"]=1 且 bomb_placed_count 未增加 |
| 无效 ignite | action["ignite"]=1 且 remote_queue 长度无变化 |
| 撞墙 | 有方向输入但对应轴坐标无变化 |
| 撞对手 | 有方向输入但坐标未变 + 被对方阻挡 |

## 五、参数可配置性

所有奖励项的值、权重、阈值均在 `RewardFunction` 子类的 `__init__` 中作为可配置参数存在，通过 `reward_config` 字典传入：

```python
class Phase1Reward(RewardFunction):
    def __init__(self, config: dict = None):
        cfg = {
            # Phase 1.1 基础奖励
            "reward_approach": 0.1,           # 曼哈顿距缩小 / 格（窗口平均后）
            "reward_approach_window": 10,     # 窗口帧数（每 N 帧评估一次平均位置推进）
            "penalty_retreat": 0.02,          # 曼哈顿距增大 / 格（弱化至原 40%）
            "penalty_center_dev": 0.04,       # 中线偏离系数
            "penalty_stall_threshold": 30,    # 无推进判定帧数阈值
            "penalty_stall_init": 0.02,       # 停滞初始帧惩罚
            "penalty_stall_cap": 0.5,         # 停滞惩罚帧上限
            "penalty_wall": 0.03,             # 撞墙 / 轴
            "penalty_blocked": 0.02,          # 撞对手
            "reward_survive": 0.001,          # 存活 / 帧 — 远小于主动奖励（1 数量级差）
            "penalty_illegal_bomb_cap": 0.1,  # 已达上限仍按 action
            "penalty_illegal_place": 0.05,    # 脚下不可放弹按 action
            "penalty_illegal_ignite": 0.05,   # 无效 ignite
            "penalty_illegal_dir": 0.05,      # 超 2 方向
            "penalty_death_self": 1.0,        # 自炸死（Phase 1.1）
            "penalty_death_opp": 0.5,         # 被对手炸死（Phase 1.1）
            # Phase 1.2 死亡惩罚提升 3 倍
            "penalty_death_self_bomb": 3.0,   # Phase 1.2+ 自炸死（3×）
            "penalty_death_opp_bomb": 1.5,    # Phase 1.2+ 被对手炸死（3×）
            # Phase 1.2 新增
            "reward_place_bomb": 0.1,         # 成功放置炸弹
            "reward_destroy_brick_fwd": 0.5,  # 炸毁向对手路上的砖块/块
            "reward_destroy_brick_side": 0.1, # 炸毁非对手方向的砖块/块（副产物）
            "penalty_bomb_wasted": 0.2,       # 炸弹空爆（0 砖块被毁）
            # Phase 1.3 新增
            "reward_pickup_normal": 0.2,      # 拾取常规 Buff
            "reward_pickup_unknown": 0.3,     # 拾取未知道具
        }
        if config:
            cfg.update(config)
        self.cfg = cfg
```

## 六、各奖励项量级估测

基于默认游戏参数（19×11地图、FPS=24、速度 2.5格/s、BOMB_FUSE=48帧），以一个完整回合（~3600帧≈150秒上限，实际典型 600~1800帧）为计算窗口。

### Phase 1.1 量级

| 奖励项 | 公式 | 单帧典型值 | 回合总和（典型） | 回合总和（极限） | 计算依据 |
|--------|------|-----------|----------------|----------------|----------|
| 🟢 曼哈顿距缩小(窗口) | +0.1 × Δavg_dist / 10帧 | 每10帧 +0.02~+0.1 | +1.5 ~ +2.5 | +2.8 | 窗口化平均，消除振荡刷分。总量不变但分配更集中 |
| 🔴 曼哈顿距增大 | -0.02 × Δdist_man | -0.001 ~ -0.002 | -0.2 ~ -0.8 | -0.54 | 减弱至原 40%，接受导航正常波动 |
| 🔴 中线偏离 | -0.04 × (dist/20)² | dist=2: -0.0004; dist=5: -0.0025; dist=10: -0.01 | -1.5 ~ -10 | -144 (极限) | dist ∈ [0,20]; 熟练 agent 偏离 <4px, 新手可能持续 >8px |
| 🔴 无推进惩罚 | max(累计,-0.5)/帧 | 0（有推进时）~ -0.02→-0.50 | 0 ~ -35 | -35 | 30帧宽限后 -0.02→-0.50，60帧停滞 = -6.5, 120帧 = -29.5+ |
| 🟢 存活 | +0.001/帧 | +0.001 | +0.6 ~ +1.8 | +3.6 | 纯背景信号，不再主导正收益 |
| 🔴 撞墙 | -0.03/事件 | 0 ~ -0.06 | -0.1 ~ -0.5 | -3.0 | 熟练 agent 几乎无碰撞，新手每帧可触发多次 |
| 🔴 非法操作 | -0.1 / -0.05 | 0（学会后）~ -0.15 | 0 ~ -50+ | -180+ | 持续 spamming 时会严重惩罚，agent 会快速学会避免 |
| 🔴 自炸/被炸 | -1.0 / -0.5 | 一次性 | 0 ~ -1.0 | -1.0 | Phase 1.1 不放炸弹，极少触发 |

### Phase 1.2 增量量级

死亡惩罚提升 3 倍（自炸 -3.0，被炸 -1.5）。新增方向性砖块奖励 + 空爆惩罚。

| 奖励项 | 公式 | 单帧典型值 | 回合总和（典型） | 回合总和（极限） | 计算依据 |
|--------|------|-----------|----------------|----------------|----------|
| 🟢 放置炸弹 | +0.1/次 | 约每 48帧 1次 (bomb_fuse) | +2.0 ~ +5.0 | +7.5 | 上限 75次（3600÷48），实际受制于行动时机 |
| 🟢 炸砖（向对手路）| +0.5/块 | — | +6 ~ +18 | +35 | ~70 砖，约 60~70% 在对手方向侧 |
| 🟡 炸砖（非对手向）| +0.1/块 | — | +0.3 ~ +0.7 | +3 | ~30% 砖在侧/反方向（边缘或身后） |
| 🔴 炸弹空爆惩罚 | -0.2/次 | — | 0 ~ -4 | -15 | 初期/砖块耗尽后频繁空爆，熟练后可避免 |
| 🔴 自炸（3×）| -3.0/次 | 一次性 | -3.0 | -3.0（1次/回合）| 每死一次 -3.0，强制珍惜生命 |

### Phase 1.3 增量量级

| 奖励项 | 公式 | 单帧典型值 | 回合总和（典型） | 回合总和（极限） | 计算依据 |
|--------|------|-----------|----------------|----------------|----------|
| 🟢 拾取 Buff | +0.2~+0.3/次 | 约每 150~300帧 1次 | +0.4 ~ +2.4 | +4.5 | 砖块掉落 ~10.5次/回合 ×0.15，定时刷新 ~5次/回合（30s间隔） |

### Phase 1.1 典型回合净奖励汇总

| 场景 | 奖励总和 | 构成 |
|------|---------|------|
| 完美Agent（推进到底、几乎零偏离、零碰撞） | +3.0 ~ +5.5 | 存活 +0.6~1.8 + 窗口推进 +2~2.8 - 偏离 -0.5 - 后退 -0.01 |
| 一般Agent（偶尔偏离、少量碰撞、偶有停滞） | -0.1 ~ -4.0 | 存活 +0.6~1.2 + 窗口推进 +1~2 - 偏离 -1~-3 - 停滞 -1~-3 - 后退 -0.2 |
| 新手Agent（频繁偏轴、常撞墙、停滞多） | -20 ~ -50 | 存活 +0.6~1.2 - 偏离 -10~-23 - 停滞 -10~-30 - 碰撞 -1~-3 |

存活奖励为纯背景信号（~总正收益的 15~25%），窗口化推进奖励和负惩罚（偏离、停滞）构成主要驱动。

## 七、已知风险与缓解

| 风险 | 缓解方案 |
|------|----------|
| Agent 学到"反复横跳"振荡刷分 | 窗口化（10帧）平均位置奖励 + 后退惩罚减至原 40%，破坏微循环；停滞惩罚进一步抑制 |
| Agent 放完炸弹就跑角落 | 放炸弹收益(+0.1)远小于"远离对手"的惩罚；空爆惩罚进一步抑制无效放弹 |
| Agent "乐不思蜀"重复刷放弹奖励 | 砖块总量 ~70 一次性消耗，且越靠近对手方向的砖收益越高(+0.5 vs +0.1)；砖块耗尽后空爆惩罚(-0.2)鼓励推进而不是原地放弹 |
| 撞墙 + 中线偏离 双罚叠加过渡惩罚 | 惩罚值均为轻量级，叠加后最大 -0.07/帧，仍在合理范围 |
| 砖块炸完后的探索真空期 | "向对手走"为主信号，不受砖块数量影响 |
